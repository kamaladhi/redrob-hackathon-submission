# Redrob Hackathon: Intelligent Candidate Discovery & Ranking

**Team:** Neural Recruit  
**Member:** Jeevakamal K R

This repository contains our submission for the Redrob Intelligent Candidate Discovery & Ranking Challenge.

## Architecture

To meet the strict compute constraints (5 minutes max runtime, CPU-only, no network access), our solution employs a two-phase architecture:

1.  **Offline Pre-computation (`build_index.py`)**: We iterate over the full 100,000 candidate dataset, extracting features, detecting honeypots, and generating dense vector embeddings using the `all-MiniLM-L6-v2` Sentence-Transformers model. We build a highly optimized FAISS index and save the candidate metadata.
2.  **Fast Ranking (`rank.py`)**: The script loads the pre-computed FAISS index and candidate metadata. It computes the semantic similarity of candidates against the Job Description. The base score is modified using a hybrid algorithm that applies heuristics for years of experience, specific domain expertise (penalizing non-tech roles), and behavioral signals (`redrob_signals`). Finally, a deterministic factual templating system generates reasoning sentences without the latency of an LLM.

## Reproducing the Submission

To reproduce our `submission.csv`, ensure you have the required dependencies installed:

```bash
pip install -r requirements.txt
```

Place the dataset (`candidates.jsonl`) in the root of the repository, and run the ranking script:

```bash
python rank.py --candidates ./candidates.jsonl --out ./submission.csv
```

*Note: The repository already contains our pre-computed `candidate_index.faiss` and `candidate_metadata.pkl` generated by the offline pipeline.*

## Docker Sandbox Validation

We have provided a `Dockerfile` that packages our environment. To test the code in a Docker sandbox against a sample candidate subset, simply build and run the image:

```bash
docker build -t redrob-sandbox .
docker run redrob-sandbox
```
